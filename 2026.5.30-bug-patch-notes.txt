THE BROKEN LOGIC PATH
---------------------

The refactor split gameLoop.ts from a monolith into modular files (types.ts,
rotation.ts, vexAlignment.ts, etc.). In the process, these lines were added to
gameLoop.ts:

    export type { ActivePiece, GhostPiece, VexSpell, GameStatus, GameState } from './types';

The types were re-exported but never locally imported. Every function signature
using GameState, ActivePiece, or GhostPiece was referencing names not in scope.
20 function signatures broke silently under tsc --noEmit.

ROOT CAUSE
----------

gameLoop.ts re-exports types from ./types without a local import type.
TypeScript's strict: true rejects using undeclared names in emit declarations.
npm run build runs tsc && vite build — since tsc failed, Vite never ran, and
the dist was never regenerated after the refactor.

Evidence:
  - dist/assets/index-P78-I1c7.js timestamp: May 23 (epoch 1779728983)
  - Source gameLoop.ts timestamp: May 25 (epoch 1779874941)
  - THE DIST WAS 2 DAYS OLDER THAN THE SOURCE — the deployed game was running pre-refactor code

Vitest tests passed because vitest uses esbuild for transforms (doesn't run tsc),
so the type-level break was invisible to the test runner. The game logic itself
was actually correct — the bug was purely a BUILD-TIME BREAK that prevented a
new dist from being generated.

FIX APPLIED
-----------

1. src/engine/gameLoop.ts — added local type imports before the re-export block:

      // Local type imports (used in function signatures)
      import type {
        ActivePiece,
        GhostPiece,
        GameState,
      } from './types';

2. src/tests/integration.test.ts — fixed 4 ancillary type errors:

      - ColorId type added to import type and used in fillLogicalRow()/lockCell() default params
      - Destructured for (const [a, b] of pairs) replaced with indexed pair[0]!, pair[1]!
        to satisfy noUncheckedIndexedAccess

CONCLUSION
----------
The game logic was never broken — the build pipeline was. npm run build was dead
(the tsc step failed before Vite ever ran), so the dist was stale for 2 days.
Vite dev and vitest worked fine because both skip tsc type-checking. After fixing
the missing import type and ancillary test errors, the build chain is healthy
and all 321 tests pass.
